# Banana Flow - AI图像处理工作流使用说明

## 项目简介

Banana Flow 是一个基于 React 和 Vite 构建的 AI 图像处理工作流应用，集成了 Google Gemini AI 和多种图像处理功能。通过直观的拖拽式节点编辑器界面，用户可以创建复杂的图像处理工作流。

## 核心功能特性

### 🎨 可视化节点编辑器
- **拖拽式界面**：通过拖拽节点创建复杂的工作流
- **实时连接**：节点间通过连接线传递数据
- **智能对齐**：自动对齐辅助线帮助精确布局
- **撤销/重做**：完整的操作历史管理

### 🤖 AI 智能处理
- **Gemini AI 集成**：基于 Google Gemini 2.5 Flash 模型
- **智能图像分析**：自动识别图像内容并生成描述
- **文本生成**：根据提示词生成文本内容
- **图像编辑**：基于 AI 提示进行图像修改

### 📁 文件管理
- **多格式支持**：支持图片、视频等多种文件格式
- **拖拽上传**：支持拖拽文件到节点进行上传
- **历史记录**：自动保存处理历史，支持快速访问
- **文件预览**：实时预览上传和处理结果

### 🔄 工作流执行
- **实时处理**：节点状态实时更新，显示处理进度
- **错误处理**：完善的错误提示和恢复机制
- **静音功能**：可临时禁用节点，跳过处理步骤
- **批量处理**：支持多个文件同时处理

### 侧边栏功能

#### 核心节点 (Core)
- **Text Input**：文本输入节点，用于输入提示词或文本内容
- **Image Input**：图像输入节点，用于上传和处理图像文件
- **Text Generator**：文本生成器，基于 AI 生成文本内容
- **Image Editor**：图像编辑器，使用 AI 进行图像修改
- **Video Generator**：视频生成器，创建视频内容
- **Output Display**：输出显示节点，展示最终结果
- **Conditional**：条件节点，实现条件分支逻辑

#### 预设模板 (Presets)
- **图片变手办**：将照片转换为手办模型效果
- **切换人物视角**：改变人物照片的拍摄角度
- **建筑图转模型**：将建筑照片转换为模型效果
- **组合物体**：将多个物体组合在一起
- **高清修复**：提升图像分辨率和质量
- **图转线稿**：将图像转换为线稿风格
- **按照色卡上色**：根据色卡为图像上色
- **更多预设**：持续更新的预设模板

## 详细使用指南

### 1. 环境配置

#### 安装依赖
```bash
npm install
```

#### 配置 API 密钥
创建 `.env.local` 文件并添加：
```
GEMINI_API_KEY=your_gemini_api_key_here
```

#### 启动应用
```bash
# 仅启动前端
npm run dev

# 仅启动后端
npm run server

# 同时启动前后端
npm run dev:full
```

### 2. 创建工作流

#### 步骤 1：添加输入节点
1. 从侧边栏拖拽 **Image Input** 节点到画布
2. 点击节点上传图像文件
3. 支持拖拽文件到节点进行快速上传

#### 步骤 2：添加处理节点
1. 从侧边栏选择所需的处理节点
2. 拖拽到画布合适位置
3. 使用预设模板可快速配置常用功能

#### 步骤 3：连接节点
1. 点击源节点的输出端口（右侧圆点）
2. 拖拽到目标节点的输入端口（左侧圆点）
3. 连接线会自动显示数据流向

#### 步骤 4：配置节点参数
1. 点击节点进行选中
2. 在节点内部输入必要的参数
3. 对于 AI 节点，输入提示词或选择预设

#### 步骤 5：执行工作流
1. 点击顶部工具栏的 **Run Workflow** 按钮
2. 观察节点状态变化（IDLE → PROCESSING → COMPLETED）
3. 在输出节点查看最终结果

### 3. 节点类型详解

#### 输入节点
- **Text Input**：
  - 用途：输入文本提示词或内容
  - 配置：直接在文本框中输入
  - 输出：文本数据

- **Image Input**：
  - 用途：上传图像文件
  - 支持格式：JPG, PNG, GIF, WebP 等
  - 输出：图像数据

#### 处理节点
- **Text Generator**：
  - 用途：基于提示词生成文本
  - 输入：文本提示词
  - 输出：生成的文本内容

- **Image Editor**：
  - 用途：使用 AI 编辑图像
  - 输入：图像 + 编辑提示词
  - 输出：编辑后的图像 + 描述文本

- **Video Generator**：
  - 用途：生成视频内容
  - 输入：图像（可选）+ 文本提示词
  - 输出：生成的视频文件

- **Conditional**：
  - 用途：条件分支处理
  - 输入：图像 + 条件提示词
  - 输出：根据条件返回不同结果

#### 输出节点
- **Output Display**：
  - 用途：显示最终处理结果
  - 功能：预览、下载、裁剪、压缩
  - 支持：图像、视频、文本等多种格式

### 4. 高级功能

#### 节点管理
- **选择节点**：点击节点进行单选，Ctrl+点击多选
- **移动节点**：拖拽节点到新位置
- **删除节点**：选中节点后按 Delete 键
- **静音节点**：右键节点选择静音，跳过处理

#### 画布操作
- **缩放**：使用鼠标滚轮或 Ctrl+滚轮
- **平移**：拖拽空白区域移动画布
- **对齐辅助**：拖拽节点时显示对齐线
- **全选**：Ctrl+A 选择所有节点

#### 历史管理
- **撤销**：Ctrl+Z 或点击撤销按钮
- **重做**：Ctrl+Y 或点击重做按钮
- **保存**：Ctrl+S 保存当前工作流
- **打开**：Ctrl+O 打开已保存的工作流

#### 文件处理
- **批量上传**：支持同时上传多个文件
- **格式转换**：自动处理不同图像格式
- **质量优化**：内置图像压缩和优化功能
- **裁剪工具**：精确裁剪图像区域

### 5. 预设模板使用

#### 快速开始
1. 从侧边栏选择预设模板
2. 拖拽到画布，自动创建完整工作流
3. 上传输入文件
4. 点击运行即可获得结果

#### 自定义预设
1. 创建基础工作流
2. 配置节点参数
3. 保存为自定义预设
4. 后续可快速复用

### 6. 故障排除

#### 常见问题
- **API 密钥错误**：检查 `.env.local` 文件配置
- **节点连接失败**：确保数据类型匹配
- **处理超时**：检查网络连接和 API 配额
- **文件上传失败**：检查文件格式和大小限制

#### 错误处理
- 节点状态显示为 ERROR 时，查看错误信息
- 使用撤销功能回退到正常状态
- 检查输入数据是否完整
- 重新配置有问题的节点

### 7. 最佳实践

#### 工作流设计
1. **从简单开始**：先创建基础工作流，再逐步复杂化
2. **合理命名**：为节点设置有意义的标签
3. **模块化设计**：将复杂流程分解为多个简单步骤
4. **测试验证**：使用小文件测试工作流效果

#### 性能优化
1. **合理使用静音**：跳过不需要的节点
2. **批量处理**：一次处理多个文件
3. **缓存结果**：保存中间结果避免重复计算
4. **网络优化**：确保稳定的网络连接

#### 数据管理
1. **定期保存**：及时保存工作流配置
2. **版本控制**：为重要工作流创建备份
3. **清理历史**：定期清理不需要的历史记录
4. **文件组织**：合理组织输入和输出文件

## 技术规格

### 系统要求
- **Node.js**：18.0 或更高版本
- **浏览器**：Chrome 90+, Firefox 88+, Safari 14+
- **内存**：建议 4GB 以上
- **存储**：至少 1GB 可用空间

### 支持格式
- **图像**：JPG, PNG, GIF, WebP, BMP
- **视频**：MP4, WebM, MOV
- **文本**：TXT, MD, JSON

### API 限制
- **Gemini API**：遵循 Google 的 API 使用限制
- **文件大小**：单个文件不超过 10MB
- **并发处理**：同时处理不超过 5 个节点
